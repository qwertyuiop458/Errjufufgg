# J2ME Reverse Explorer

Веб-приложение для реверс-инжиниринга Java ME (J2ME) игр из `.jar` и `.jad`.

## Возможности

- Загрузка JAR и разбор на интерактивные категории: графика, музыка, классы, текст, спрайты, палитры.
- Поддержка JAD с извлечением метаданных (`MIDlet-Name`, `MIDlet-Version`, `MIDlet-Jar-URL` и т.д.).
- Для JAD можно добавить companion JAR, чтобы сразу увидеть внутренности игры.
- Предпросмотр изображений, аудио и текстовых ресурсов прямо в браузере.
- Поиск по путям файлов внутри архива.
- Скачивание любого извлечённого артефакта.
- Декомпиляция `.class` через endpoint `GET /decompile/<session_id>/<path:raw_path>`.

## Запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Откройте `http://localhost:8000`.

## Настройка декомпиляции

### 1) Java Runtime

Для декомпиляции нужен установленный Java Runtime (`java` в `PATH`):

```bash
java -version
```

### 2) JAR декомпилятора

Положите один из jar-файлов в папку `tools/`:

- Вариант A (по умолчанию): `tools/cfr.jar`
- Вариант B (fallback): `tools/fernflower.jar`

Приложение сначала пытается использовать CFR, если `tools/cfr.jar` найден, иначе FernFlower.

При необходимости пути можно переопределить переменными окружения:

- `DECOMPILER_CFR_JAR`
- `DECOMPILER_FERNFLOWER_JAR`
- `JAVA_BIN` (по умолчанию `java`)
- `DECOMPILER_TIMEOUT` (в секундах, по умолчанию `15`)

## Ограничения декомпиляции

- Обфусцированные/повреждённые классы могут декомпилироваться частично или не декомпилироваться совсем.
- Результат декомпиляции не эквивалентен оригинальным исходникам и может отличаться по именам переменных и структуре.
- Для некоторых классов декомпилятор может завершиться по таймауту.
